1 article peut être 

- le résumé d'un ouvrage
- un article d'opinion
- la description d'une technique
- une analyse thématique
- ...

--> classement par catégories

dans le cas d'un résumé, il faut indiquer le nom de l'auteur original et la référence complète
par ailleurs, un article présenté sur Knine peut aussi avoir fait l'objet d'une ou plusieurs publications

--> utilisation d'un objet Publication pour stocker ces informations

il faut pouvoir déduire si l'article est une oeuvre originale ou qui se rapporte à une autre (résumé, critique, analyse, commentaires, ...)

champ 'type' : selection original | summary | review | analysis | comments
champ 'publications_ids' : many2many

tous les articles ont 
- un ou plusieurs auteurs
- une date de publication
- un ou plusieurs labels (catégorisation hiérarchique)


un article peut comporter une ou plusieurs références (notes de bas de page ou référenes à d'autres ouvrages)

champ 'references_ids' : one2many

Reference
---------
notes		text
publication	many2one



Publication 
-----------

authors_ids	many2many
type		selection : article | book
title		string (le titre de l'oeuvre originale)
publisher	string	(la maison d'édition pour un livre, le nom de la revue pour un article, nom d'une association ou d'un site web)
place		string  (ville de publicaition ou URL complète dans le cas d'une publication web)
year		integer (année de publicaiton)
number		integer (le cas échéant, le numéro de la revue)
pages		string  (le nombre de pages pour un livre, les pages concernées de la revue pour un article)



Author
------
firstname	string
lastname	string
